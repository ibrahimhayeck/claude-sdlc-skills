# Issue tracker: Jira

Work for this repo is tracked in Jira.

- **Project key:** `<KEY>`
- **Site:** `<https://your-org.atlassian.net>`
- **Repo label:** `repo:<repo-name>` — apply to every issue so multi-repo epics can be
  filtered by repo (only relevant if work spans repos).
- **Access method:** `<Atlassian MCP connector | acli/REST with token | draft-only>`

## How skills use Jira
- `to-prd` — creates an **epic** in `<KEY>`, applies `needs-triage`, and links to the PRD
  document (stored per `docs/agents/docs-location.md`).
- `to-issues` — creates one **story** per vertical slice under the epic, in dependency order.
  For a multi-repo epic, one story per repo per slice, each labelled `repo:<name>`.
- `triage` — moves issues through the state labels/statuses in `triage-labels.md`. Every
  comment the agent posts during triage starts with `> *This was generated by AI during triage.*`

## If access is "draft-only"
The agent writes the epic/story text into the chat (or a local `.scratch/` file) with the
target project and labels noted; a human creates the ticket. Keep the Jira key in the PRD
once created so links stay intact.
