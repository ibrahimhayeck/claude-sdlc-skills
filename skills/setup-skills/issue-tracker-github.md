# Issue tracker: GitHub Issues

Work for this repo is tracked in GitHub Issues.

- **Repo:** `<owner/name>`
- **Access:** the `gh` CLI (`gh issue create`, `gh issue list`, `gh issue edit`) or the GitHub MCP.

## How skills use it
- `to-prd` — creates an issue holding the PRD (or links to the PRD doc — see
  `docs/agents/docs-location.md`), labelled `needs-triage`.
- `to-issues` — creates one issue per vertical slice, in dependency order, referencing real
  issue numbers in "Blocked by".
- `triage` — applies the state/category labels from `triage-labels.md`. Every triage comment
  starts with `> *This was generated by AI during triage.*`
